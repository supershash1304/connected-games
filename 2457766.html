Week 3-4

Taking upon the server side implementation task 1. Unity Netcode for GameObjects (NGO)
* Purpose: Provides a high-level API for implementing multiplayer games.
* Installation:
* Open Unity's Package Manager (Window > Package Manager).
* Search for "Netcode for GameObjects" and install it.
* Use Case: Handles networking, synchronization, and player management.

2. Unity Transport Package
Purpose: Provides low-level networking transport for Netcode for GameObjects.
Installation:
In the Package Manager, search for "Unity Transport" and install it.
Use Case: Manages network connections and data transmission.

3. Unity Multiplayer Tools (Optional)
* Purpose: Provides additional tools for debugging and testing multiplayer games.
* Installation:
* In the Package Manager, search for "Multiplayer Tools" and install it.
* Use Case: Helps with network simulation, latency testing, and debugging.

Implementing the multiplayer server creation process

1. Set Up the multiplayer UI


Create a Canvas:
Right-click in the Hierarchy > UI > Canvas.
This will automatically create an EventSystem as well.
Add Buttons:
Right-click on the Canvas > UI > Button.
Create three buttons:
Host Button: For starting the game as both the host and a player.
Server Button: For starting the game as a dedicated server.
Client Button: For joining the game as a client.
Label the Buttons:
Select each button and change its text (in the Text child object) to "Host", "Server", and "Client".
Position the Buttons:
Use the RectTransform tool to position the buttons neatly on the screen.

2.Assign the Script to the UI
Attach the Script:
Select the Canvas in the Hierarchy.
Add the MultiplayerUI script as a component.
Assign Buttons:
Drag the corresponding buttons from the Hierarchy into the hostButton, serverButton,
and clientButton fields in the MultiplayerUI script.
